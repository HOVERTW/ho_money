# 📤 手動上傳與增強同步功能使用指南

## 🎯 功能概述

我們已經成功實現了完整的雲端同步功能，包括：
1. **按鍵式手動上傳** - 一次性上傳所有本地數據到雲端
2. **實時增強同步** - 自動同步所有的創建、更新、刪除操作

## 🔧 實現的功能

### 1. 手動上傳服務
- **文件位置**: `src/services/manualUploadService.ts`
- **功能**: 專門處理本地數據到 Supabase 的一次性上傳
- **支援數據類型**:
  - 交易記錄 (transactions)
  - 資產數據 (assets)
  - 負債數據 (liabilities)
  - 帳戶數據 (accounts)
  - 交易類別 (categories) ✨ **新增**

### 2. 增強同步服務 ✨ **新功能**
- **文件位置**: `src/services/enhancedSyncService.ts`
- **功能**: 實時同步所有的創建、更新、刪除操作到雲端
- **支援操作**:
  - 資產的更新和刪除同步
  - 負債的更新和刪除同步
  - 交易的更新同步
  - 類別的更新和刪除同步

### 3. 儀表板上傳按鈕
- **位置**: 儀表板右上角
- **顯示條件**: 只在用戶已登錄時顯示
- **圖標**: 雲端上傳圖標 (cloud-upload-outline)
- **功能**: 點擊後觸發手動上傳流程

### 4. 實時同步集成 ✨ **新功能**
- **自動觸發**: 所有數據操作後自動執行
- **無感知同步**: 用戶無需手動操作
- **錯誤處理**: 失敗時不影響本地操作
- **日誌記錄**: 詳細的同步狀態日誌

### 5. 數據處理特性
- **UUID 自動生成**: 自動為無效 ID 生成有效的 UUID
- **數據驗證**: 過濾掉無效的交易記錄
- **錯誤處理**: 詳細的錯誤報告和處理
- **進度反饋**: 顯示上傳進度和結果

## 🚀 使用方法

### 步驟 1: 登錄帳號
1. 打開應用
2. 點擊「體驗雲端同步」或登錄按鈕
3. 使用 Google 登錄或電子郵件註冊/登錄

### 步驟 2: 手動上傳
1. 登錄成功後，在儀表板右上角會出現上傳按鈕
2. 點擊上傳按鈕 (雲端圖標)
3. 系統會顯示「上傳中」提示
4. 等待上傳完成

### 步驟 3: 查看結果
上傳完成後會顯示詳細結果：
- ✅ **成功**: 顯示各類型數據的上傳數量
- ❌ **失敗**: 顯示錯誤詳情和失敗原因

## 🔄 實時同步功能 ✨ **新功能**

### 自動同步的操作
- **資產管理**: 更新資產價值、刪除資產 → 自動同步到雲端
- **負債管理**: 更新負債餘額、刪除負債 → 自動同步到雲端
- **交易管理**: 修改交易記錄 → 自動同步到雲端
- **類別管理**: 編輯類別、刪除類別 → 自動同步到雲端

### 同步時機
- **即時同步**: 操作完成後立即執行
- **靜默同步**: 後台執行，不干擾用戶操作
- **失敗容錯**: 同步失敗不影響本地操作

## 🔒 安全機制

### 1. 用戶驗證
- 只有已登錄用戶才能上傳數據
- 使用 Supabase 的認證系統

### 2. Row Level Security (RLS)
- 每個用戶只能訪問自己的數據
- 防止數據洩露和未授權訪問

### 3. 數據隔離
- 自動添加 `user_id` 到所有記錄
- 確保數據歸屬正確

## 📝 使用注意事項

### 1. 網路連接
- 確保設備有穩定的網路連接
- 上傳大量數據時可能需要較長時間

### 2. 數據重複
- 使用 `upsert` 操作避免重複數據
- 相同 ID 的記錄會被更新而非重複插入

### 3. 登錄狀態
- 必須保持登錄狀態才能上傳
- 登錄過期需要重新登錄
